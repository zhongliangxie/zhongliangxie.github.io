自动让Unity复制文件到Android APK的'assets'目录
在“Assets/Plugins/Android”中创建一个'assets'目录，让Unity自动将所有包含的文件复制到APK的'assets'目录中。
2021-07-07 17:31 folder 游戏开发   label Unity  

解决方案

我最近不得不将一个 SDK 集成到一个使用 Unity 3D 制作的 Android 应用程序中。我不得不将一些配置文件直接放入 APK 内的“资产”目录中。很自然地，Unity 将我的游戏打包成所需的 APK 格式。但是如果我想改变内容呢？我必须使用APK Tool解压缩它并添加我的文件。然后重新打包。

一定会有更好的办法。我认为很多游戏开发者都必须做这种事情，而 Unity 会提供一种简单的方法。不幸的是，文档只提到了 Plugins 目录。这对于包含本机库和二进制文件非常有用。但是 Plugins 目录内置了更多的秘密功能。您可以直接在 Plugins 目录中创建一个“assets”目录。Unity 会将内容复制到 APK。很好用，但没有文档。

'Assets/StreamingAssets'目录也可用于让 Unity 将文件复制到 Android ‘assets’ 目录。我看到的唯一区别是“Assets/StreamingAssets”中的文件会在每个平台上复制，而 'Assets/Plugins/Android/assets' 中的文件仅在构建 Android 时复制。这两种方法都是可行的选择，具体取决于您想要实现的目标。

这只是一个小技巧。但我想分享它。当然，您可以导出一个Android Studio项目并在其中添加资产。
